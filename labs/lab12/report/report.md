# Цель работы

Приобретение практических навыков по настройке доступа локальной сети к внешней сети посредством NAT.

# Теоретические сведения

**NAT (Network Address Translation)** — это технология преобразования сетевых адресов, применяемая на маршрутизаторах для изменения IP-адресов в пакетах при передаче между сетями (обычно между локальной сетью и Интернетом).

Основная идея NAT заключается в том, что устройства внутри локальной сети используют **частные IP-адреса**, а при выходе во внешнюю сеть эти адреса заменяются на **публичный IP-адрес** маршрутизатора или адрес из специального пула.

**Основные функции NAT:**
– преобразование внутренних IP-адресов во внешние;
– обратное преобразование для входящих пакетов;
– обеспечение связи между частной и публичной сетями.

**Преимущества NAT:**
– экономия публичных IP-адресов;
– возможность подключения большого числа устройств к Интернету;
– скрытие внутренней структуры сети (повышение безопасности).

**Основные виды NAT:**
– **Static NAT** — фиксированное соответствие одного внутреннего IP одному внешнему;
– **Dynamic NAT** — динамическое сопоставление адресов из пула;
– **PAT (NAT overload)** — использование одного IP-адреса для множества устройств за счёт разных портов.

**Принцип работы:**
При отправке пакета во внешнюю сеть маршрутизатор:

1. заменяет внутренний IP-адрес на внешний;
2. сохраняет соответствие в таблице NAT;
3. при получении ответа выполняет обратное преобразование.

**Где настраивается:**
NAT настраивается на граничном маршрутизаторе между локальной и внешней сетью с указанием:
– внутренних (inside) и внешних (outside) интерфейсов;
– правил трансляции (ACL, пулы адресов или статические соответствия).

# Контрольные вопросы

1. **В чём состоит основной принцип работы NAT (что даёт наличие NAT в сети организации)?**  

   **NAT (Network Address Translation)** — это технология преобразования IP-адресов, при которой внутренние (частные) IP-адреса сети заменяются на внешний (публичный) адрес при выходе в Интернет.  

   Основной принцип работы заключается в том, что маршрутизатор изменяет IP-адрес источника пакета при его отправке во внешнюю сеть и выполняет обратное преобразование при получении ответа.  

   Использование NAT даёт следующие преимущества:
   – позволяет использовать частные IP-адреса внутри организации;  
   – экономит публичные IP-адреса;  
   – повышает безопасность сети, скрывая внутреннюю структуру;  
   – обеспечивает доступ нескольких устройств к Интернету через один внешний адрес.

2. **В чём состоит принцип настройки NAT (на каком оборудовании и что нужно настроить для выхода из локальной сети во внешнюю сеть через NAT)?**  

   NAT настраивается на маршрутизаторе, который соединяет локальную сеть с внешней сетью (Интернетом).  

   Для настройки необходимо:
   – определить внутренние и внешние интерфейсы (`ip nat inside` и `ip nat outside`);  
   – создать список доступа (ACL), определяющий, какие адреса будут транслироваться;  
   – при необходимости создать пул внешних IP-адресов;  
   – настроить правило трансляции (например, `ip nat inside source list ... pool ... overload`).  

   После этого маршрутизатор начинает выполнять преобразование адресов для трафика, выходящего во внешнюю сеть.

3. **Можно ли применить Cisco IOS NAT к субинтерфейсам?**  

   Да, NAT можно применять к субинтерфейсам.  

   В Cisco IOS субинтерфейсы (например, FastEthernet0/0.1, FastEthernet0/0.2) могут быть настроены как внутренние или внешние интерфейсы NAT. Это особенно актуально при использовании VLAN и технологии Router-on-a-Stick, где каждый VLAN представлен отдельным субинтерфейсом.

4. **Что такое пулы IP NAT?**  

   **Пул NAT** — это диапазон публичных IP-адресов, который используется для динамической трансляции внутренних адресов.  

   Когда внутренние устройства выходят в Интернет, маршрутизатор назначает им свободный адрес из пула. При использовании режима overload (PAT) несколько устройств могут использовать один IP-адрес, различаясь по номерам портов.

5. **Что такое статические преобразования NAT?**  

   **Статическое NAT-преобразование** — это постоянное соответствие между внутренним и внешним IP-адресом или портом.  

   Оно используется для обеспечения доступа к внутренним серверам из внешней сети. Например, можно настроить правило, при котором внешний адрес и порт (например, 198.51.100.2:80) всегда перенаправляются на конкретный внутренний сервер.  

   Основные особенности:
   – фиксированное соответствие адресов или портов;  
   – используется для серверов (WEB, FTP, почта и т.д.);  
   – обеспечивает доступ извне к ресурсам локальной сети.
