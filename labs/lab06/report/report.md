# Цель работы

Настроить статическую маршрутизацию VLAN в сети.

# Теоретические сведения

Маршрутизация — это процесс передачи сетевых пакетов между различными сетями на основе информации о сетевых адресах. Основной задачей маршрутизатора является определение оптимального пути доставки пакета от источника к получателю. Для этого используются таблицы маршрутизации, в которых содержатся сведения о доступных сетях и интерфейсах, через которые можно передать данные.

Маршрутизаторы работают на сетевом уровне модели **OSI (Network Layer)** и анализируют заголовки IP-пакетов. На основании IP-адреса назначения устройство выбирает соответствующий маршрут и пересылает пакет через нужный интерфейс. В процессе прохождения пакета через маршрутизатор может изменяться служебная информация, например уменьшается значение **TTL (Time To Live)**, что предотвращает бесконечную циркуляцию пакетов в сети.

Инкапсуляция — это процесс последовательного добавления заголовков различных протоколов к передаваемым данным при их перемещении по уровням сетевой модели. Когда приложение передаёт данные в сеть, информация проходит через уровни модели **OSI** или **TCP/IP**, и каждый уровень добавляет собственный заголовок, содержащий служебную информацию.

На транспортном уровне формируется сегмент или датаграмма, содержащая информацию о портах отправителя и получателя. Далее на сетевом уровне создаётся **IP-пакет**, в котором указываются IP-адрес источника и назначения, а также другие параметры, необходимые для маршрутизации. После этого на канальном уровне данные помещаются в **Ethernet-кадр**, содержащий MAC-адреса отправителя и получателя.

При передаче данных между различными сегментами сети используется технология **VLAN** и протокол **IEEE 802.1Q**, который позволяет добавлять тег VLAN в заголовок Ethernet-кадра. Это обеспечивает логическое разделение сети на несколько виртуальных сегментов и повышает уровень безопасности и управляемости сети.

Если устройства находятся в разных VLAN, обмен данными между ними осуществляется через маршрутизатор или коммутатор третьего уровня. В случае использования маршрутизатора применяется схема **Router-on-a-Stick**, при которой один физический интерфейс маршрутизатора обслуживает несколько VLAN. Для этого на интерфейсе создаются виртуальные подинтерфейсы, каждому из которых назначается собственный VLAN-идентификатор и IP-адрес.

Во время передачи данных между VLAN пакет проходит несколько этапов. Сначала отправитель формирует Ethernet-кадр и передаёт его на коммутатор. Коммутатор определяет VLAN и направляет кадр через trunk-соединение к маршрутизатору. Маршрутизатор принимает кадр, удаляет заголовок канального уровня, анализирует IP-заголовок и определяет маршрут. Затем формируется новый Ethernet-кадр для сети назначения, и пакет передаётся в соответствующий VLAN.

   Данная схема настройки реализует технологию **Router-on-a-Stick**, при которой один физический интерфейс маршрутизатора обслуживает несколько VLAN посредством логических подинтерфейсов.

# Контрольные вопросы

1. **Охарактеризуйте стандарт IEEE 802.1Q.**

   **IEEE 802.1Q** — это стандарт, определяющий механизм тегирования кадров Ethernet для передачи трафика нескольких виртуальных локальных сетей (**VLAN**) по одному физическому каналу связи. Стандарт используется в коммутируемых сетях для логического разделения сети на отдельные сегменты без необходимости создания отдельных физических соединений.

   Основная задача стандарта — обеспечить возможность передачи кадров различных VLAN через одно соединение между сетевыми устройствами, например между коммутаторами или между коммутатором и маршрутизатором. Такое соединение называется **trunk-портом**.

   При использовании IEEE 802.1Q в Ethernet-кадр добавляется специальный тег VLAN, который содержит идентификатор виртуальной сети. Благодаря этому сетевые устройства могут определить, к какой VLAN относится передаваемый кадр, и направить его только в соответствующий сегмент сети.

   Основные особенности стандарта:
   – позволяет передавать трафик нескольких VLAN по одному физическому каналу;
   – используется на **канальном уровне модели OSI**;
   – добавляет специальный тег VLAN в Ethernet-кадр;
   – применяется на trunk-соединениях между коммутаторами и маршрутизаторами;
   – обеспечивает логическое разделение сети и повышает безопасность и управляемость сети.

2. **Опишите формат кадра IEEE 802.1Q.**

   Кадр **IEEE 802.1Q** представляет собой стандартный Ethernet-кадр, в который добавляется дополнительное поле — **VLAN-тег**. Этот тег вставляется между полями MAC-адреса источника и полем типа протокола Ethernet.

   Структура кадра включает следующие элементы:

   – **Preamble (Преамбула)** — последовательность битов, используемая для синхронизации устройств перед началом передачи кадра;  
   – **SFD (Start Frame Delimiter)** — поле, обозначающее начало кадра;  
   – **MAC-адрес назначения (Destination MAC Address)** — адрес устройства-получателя;  
   – **MAC-адрес источника (Source MAC Address)** — адрес устройства-отправителя;  
   – **TPID (Tag Protocol Identifier)** — поле, имеющее значение **0x8100**, указывающее на наличие тега VLAN;  
   – **TCI (Tag Control Information)** — поле управления тегом, содержащее идентификатор VLAN и служебную информацию;  
   – **Type/Length** — поле, определяющее тип протокола верхнего уровня (например IPv4);  
   – **Data** — полезные данные кадра (инкапсулированный сетевой пакет);  
   – **FCS (Frame Check Sequence)** — контрольная сумма, используемая для проверки целостности кадра.

   Поле **TCI** включает несколько частей:
   – **Priority (PCP)** — уровень приоритета кадра;
   – **DEI (Drop Eligible Indicator)** — индикатор возможности удаления кадра при перегрузке сети;
   – **VLAN ID** — идентификатор виртуальной локальной сети.
