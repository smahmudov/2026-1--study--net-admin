# Цель работы

Получить основные навыки по настройке VLAN на коммутаторах сети.

# Теоретические сведения

**VLAN (Virtual Local Area Network)** — это технология логического разделения локальной сети на несколько независимых сегментов на уровне канального уровня модели OSI. Она позволяет объединять устройства в отдельные логические сети независимо от их физического расположения.

Основная цель использования VLAN — повышение безопасности сети, уменьшение широковещательного трафика и упрощение управления сетью. Устройства, принадлежащие разным VLAN, не могут напрямую обмениваться данными без использования маршрутизации.

Каждая VLAN имеет уникальный **идентификатор VLAN ID**, который используется для определения принадлежности устройства к определённой виртуальной сети. Обычно диапазон VLAN ID составляет **от 1 до 4094**.

Для работы VLAN используются два основных типа портов коммутатора:

* **Access-порт** — порт, который принадлежит только одной VLAN. К таким портам обычно подключаются конечные устройства (компьютеры, принтеры, серверы).
* **Trunk-порт** — порт, который может передавать трафик нескольких VLAN одновременно. Такие порты используются для соединения коммутаторов между собой.

При передаче данных между коммутаторами используется стандарт **IEEE 802.1Q**, который добавляет в Ethernet-кадр специальный тег VLAN. Этот тег содержит идентификатор VLAN и позволяет коммутаторам определять, к какой виртуальной сети относится кадр.

Для централизованного управления VLAN в сети коммутаторов Cisco применяется **VLAN Trunking Protocol (VTP)**. Этот протокол позволяет автоматически распространять информацию о VLAN между коммутаторами внутри одного домена VTP.

Использование VLAN позволяет:

* логически разделять сеть на сегменты;
* уменьшать широковещательный трафик;
* повышать уровень безопасности;
* упрощать администрирование сети.

# Контрольные вопросы

1. **Какая команда используется для просмотра списка VLAN на сетевом устройстве?**

   Для просмотра списка VLAN на коммутаторе Cisco используется команда:

   **show vlan brief**

   Данная команда выводит краткую информацию о всех настроенных VLAN на устройстве. В результате отображаются следующие данные:
   – номер VLAN;  
   – имя VLAN;  
   – состояние VLAN;  
   – список портов, принадлежащих данному VLAN.

   Также могут использоваться дополнительные команды для просмотра информации о VLAN:

   **show vlan** — отображает подробную информацию о VLAN.  
   **show interfaces trunk** — показывает список trunk-портов и VLAN, передаваемых через них.  
   **show mac address-table** — позволяет увидеть таблицу MAC-адресов и их принадлежность к VLAN.

2. **Охарактеризуйте VLAN Trunking Protocol (VTP). Приведите перечень команд с пояснениями для настройки и просмотра информации о VLAN.**

   **VLAN Trunking Protocol (VTP)** — это протокол канального уровня, предназначенный для централизованного управления VLAN в сети коммутаторов Cisco. Он позволяет автоматически распространять информацию о созданных VLAN между коммутаторами, объединёнными в один VTP-домен.

   Основные режимы работы VTP:

   – **Server** — позволяет создавать, изменять и удалять VLAN, а также распространяет информацию о VLAN другим коммутаторам.  
   – **Client** — получает информацию о VLAN от VTP-сервера, но не может изменять конфигурацию VLAN.  
   – **Transparent** — не участвует в синхронизации VLAN, но может передавать VTP-сообщения другим устройствам.

   Основные команды настройки VTP:

   **vtp mode server** — переводит коммутатор в режим VTP-сервера.  
   **vtp mode client** — переводит коммутатор в режим VTP-клиента.  
   **vtp domain имя_домена** — задаёт имя VTP-домена.  
   **vtp password пароль** — устанавливает пароль для VTP-домена.  
   **vlan номер** — создаёт VLAN.  
   **name имя** — задаёт имя VLAN.

   Команды для просмотра информации:

   **show vtp status** — отображает состояние VTP, режим работы и версию протокола.  
   **show vlan brief** — выводит список VLAN на коммутаторе.  
   **show interfaces trunk** — показывает информацию о trunk-соединениях.

3. **Охарактеризуйте Internet Control Message Protocol (ICMP). Опишите формат пакета ICMP.**

   **Internet Control Message Protocol (ICMP)** — это служебный протокол сетевого уровня, используемый для диагностики и передачи сообщений об ошибках в IP-сетях. Он применяется для проверки доступности узлов сети и информирования о проблемах при передаче пакетов.

   Наиболее известное применение ICMP — команда **ping**, которая использует сообщения **Echo Request** и **Echo Reply**.

   Основные поля пакета ICMP:

   – **Type** — тип сообщения ICMP (например, Echo Request или Echo Reply);  
   – **Code** — уточняющий код сообщения;  
   – **Checksum** — контрольная сумма для проверки целостности пакета;  
   – **Identifier** — идентификатор сообщения;  
   – **Sequence Number** — номер последовательности;  
   – **Data** — данные сообщения.

   ICMP-пакет инкапсулируется внутри IP-пакета и используется для передачи служебной информации между сетевыми устройствами.

4. **Охарактеризуйте Address Resolution Protocol (ARP). Опишите формат пакета ARP.**

   **Address Resolution Protocol (ARP)** — это протокол канального уровня, предназначенный для определения MAC-адреса устройства по его IP-адресу в локальной сети.

   Когда устройство знает IP-адрес получателя, но не знает его MAC-адрес, оно отправляет широковещательный **ARP-запрос**. Устройство с соответствующим IP-адресом отправляет **ARP-ответ**, содержащий свой MAC-адрес.

   Основные поля ARP-пакета:

   – **Hardware Type (HTYPE)** — тип аппаратной сети (например, Ethernet);  
   – **Protocol Type (PTYPE)** — тип сетевого протокола (например, IPv4);  
   – **Hardware Address Length (HLEN)** — длина MAC-адреса;  
   – **Protocol Address Length (PLEN)** — длина IP-адреса;  
   – **Operation (OPER)** — тип операции (запрос или ответ);  
   – **Sender Hardware Address** — MAC-адрес отправителя;  
   – **Sender Protocol Address** — IP-адрес отправителя;  
   – **Target Hardware Address** — MAC-адрес получателя;  
   – **Target Protocol Address** — IP-адрес получателя.

   ARP позволяет сетевым устройствам корректно доставлять кадры Ethernet внутри локальной сети.

5. **Что такое MAC-адрес? Какова его структура?**

   **MAC-адрес (Media Access Control Address)** — это уникальный физический адрес сетевого интерфейса, используемый для идентификации устройства на канальном уровне модели OSI.

   MAC-адрес имеет длину **48 бит (6 байт)** и обычно записывается в шестнадцатеричном формате, например:  
   **00:1A:2B:3C:4D:5E**

   Структура MAC-адреса:

   – **OUI (Organizationally Unique Identifier)** — первые 24 бита, идентификатор производителя сетевого оборудования;  
   – **NIC Specific** — последние 24 бита, уникальный номер сетевого интерфейса, присвоенный производителем.

   MAC-адрес используется коммутаторами для определения направления передачи кадров внутри локальной сети и хранится в таблице MAC-адресов устройства.
