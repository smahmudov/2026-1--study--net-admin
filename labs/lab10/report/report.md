# Цель работы

Освоить настройку прав доступа пользователей к ресурсам сети.

# Теоретические сведения

Списки контроля доступа (Access Control Lists, ACL) — это механизм фильтрации сетевого трафика, применяемый на маршрутизаторах и коммутаторах для управления доступом между сетями. ACL позволяют разрешать или запрещать передачу пакетов на основе заданных критериев.

Основные функции ACL:
– фильтрация трафика по IP-адресам источника и назначения;  
– ограничение доступа к сетевым ресурсам;  
– управление доступом по протоколам (TCP, UDP, ICMP);  
– фильтрация по номерам портов;  
– повышение безопасности сети.

Существует два основных типа ACL:
– стандартные — фильтруют трафик только по IP-адресу источника;  
– расширенные — позволяют учитывать IP-адрес источника и назначения, протокол и номер порта.

Каждое правило ACL состоит из:
– действия (permit — разрешить, deny — запретить);  
– протокола (ip, tcp, udp, icmp);  
– адреса источника и назначения;  
– дополнительных параметров (например, номер порта).

ACL обрабатываются последовательно сверху вниз. Как только найдено совпадение с правилом, дальнейшая проверка прекращается. В конце любого списка неявно присутствует правило deny all, запрещающее весь неразрешённый трафик.

Списки контроля доступа применяются к интерфейсам устройств:
– inbound (in) — для входящего трафика;  
– outbound (out) — для исходящего трафика.

# Контрольные вопросы

1. **Как задать действие правила для конкретного протокола?**  

   Действие правила для конкретного протокола задаётся при создании записи в списке контроля доступа с указанием типа протокола. В расширенных ACL это выполняется с помощью ключевых слов, обозначающих протокол, например:
   
   – tcp — для протокола TCP;  
   – udp — для протокола UDP;  
   – icmp — для протокола ICMP;  
   – ip — для всех IP-пакетов.  

   Пример:
   permit tcp any host 10.128.0.2 eq 80  
   Данное правило разрешает передачу TCP-трафика на указанный узел через порт 80.

2. **Как задать действие правила сразу для нескольких портов?**  

   Для задания действия сразу для нескольких портов используются специальные ключевые слова:
   
   – eq — для одного конкретного порта;  
   – range — для диапазона портов;  
   – gt / lt — для портов больше или меньше заданного значения.  

   Пример:
   permit tcp any host 10.128.0.2 range 20 21  
   Данное правило разрешает доступ к портам 20 и 21 (используются протоколом FTP).

   Таким образом, с помощью диапазона портов можно задать правило сразу для нескольких сервисов.

3. **Как узнать номер правила в списке прав доступа?**  

   Номер правила можно узнать с помощью команды:
   show access-lists  

   В выводе команды каждая строка ACL имеет свой порядковый номер (sequence number), который отображается перед правилом. Этот номер используется для управления порядком правил и их редактирования.

4. **Каким образом можно изменить порядок применения правил в списке контроля доступа?**  

   Порядок применения правил в ACL определяется их последовательностью сверху вниз. Изменить порядок можно несколькими способами:
   
   – добавлением правила с указанием номера (sequence number), чтобы разместить его в нужной позиции;  
   – удалением и повторным добавлением правил в нужном порядке;  
   – использованием команды с номером строки, например:
     10 permit icmp any any  

   Важно учитывать, что маршрутизатор обрабатывает правила последовательно, и первое совпавшее правило применяется к пакету, после чего дальнейшая проверка прекращается.
