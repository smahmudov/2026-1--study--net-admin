Теоретическая часть

Назначение DHCP

  • DHCP (Dynamic Host Configuration Protocol) предназначен для автоматической выдачи сетевых параметров клиентам
  • Позволяет централизованно назначать:
    • IP-адрес
    • маску подсети
    • шлюз по умолчанию
    • DNS-сервер
  • Упрощает администрирование сети и снижает вероятность ошибок

Принцип работы DHCP

  • DHCP работает по модели клиент–сервер
  • Сервер использует пул адресов
  • Адреса выдаются на ограниченное время (lease)
  • Клиент получает параметры автоматически при подключении к сети

Обмен DHCP-сообщениями (DORA)

  • DHCP Discover
    • клиент ищет DHCP-сервер
  • DHCP Offer
    • сервер предлагает адрес и параметры
  • DHCP Request
    • клиент подтверждает выбор
  • DHCP ACK
    • сервер закрепляет адрес за клиентом
