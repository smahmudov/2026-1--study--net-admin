# Цель работы

Приобретение практических навыков по настройке динамического распределения IP-адресов посредством протокола DHCP (Dynamic Host Configuration Protocol) в локальной сети.

# Теоретические сведения

Протокол DHCP (Dynamic Host Configuration Protocol) предназначен для автоматической настройки сетевых параметров устройств в IP-сетях. Он позволяет динамически назначать IP-адреса, маску подсети, адрес шлюза по умолчанию, DNS-сервер и другие параметры без необходимости ручной конфигурации.

DHCP работает по модели клиент–сервер. В сети выделяется DHCP-сервер, который управляет пулом доступных IP-адресов и выдает их клиентским устройствам на ограниченный промежуток времени (аренду, lease). Клиенты, подключаясь к сети, автоматически запрашивают необходимые параметры.

Процесс получения IP-адреса по DHCP включает четыре основных этапа (DORA):
– DHCP Discover — клиент отправляет широковещательный запрос с целью найти DHCP-сервер;  
– DHCP Offer — сервер предлагает клиенту доступный IP-адрес и параметры сети;  
– DHCP Request — клиент подтверждает выбор предложенного адреса;  
– DHCP ACK — сервер подтверждает назначение адреса и завершает процесс конфигурации.  

На начальном этапе клиент не имеет IP-адреса, поэтому использует адрес 0.0.0.0 и отправляет запрос на широковещательный адрес 255.255.255.255. В процессе обмена используются UDP-порты 67 (сервер) и 68 (клиент).

DHCP-сервер может назначать адреса из заранее заданного диапазона (пула адресов). При этом часть адресов может быть исключена из распределения для использования в статической настройке. Также возможно резервирование адресов по MAC-адресу клиента.

Каждому выданному IP-адресу соответствует время аренды. По истечении этого времени клиент должен продлить аренду, отправив повторный запрос серверу. Если сервер недоступен, клиент может временно продолжать использовать адрес, но при длительном отсутствии подтверждения адрес освобождается.

Использование DHCP значительно упрощает администрирование сети, снижает вероятность ошибок при настройке и позволяет централизованно управлять параметрами сети.

# Контрольные вопросы

1. **За что отвечает протокол DHCP?**  

   Протокол DHCP (Dynamic Host Configuration Protocol) отвечает за автоматическое назначение сетевых параметров устройствам в сети.  

   Он позволяет динамически выдавать:  
   – IP-адрес;  
   – маску подсети;  
   – шлюз по умолчанию;  
   – адрес DNS-сервера и другие параметры.  

   Использование DHCP упрощает администрирование сети, исключает необходимость ручной настройки и снижает вероятность ошибок.

2. **Какие типы DHCP-сообщений передаются по сети?**  

   Основные типы сообщений DHCP образуют процесс DORA:  
   – **DHCP Discover** — широковещательный запрос клиента на поиск сервера;  
   – **DHCP Offer** — предложение IP-адреса от сервера;  
   – **DHCP Request** — подтверждение клиентом выбора адреса;  
   – **DHCP ACK** — окончательное подтверждение и выдача параметров.  

   Дополнительно используются:  
   – **DHCP NAK** — отказ в выдаче адреса;  
   – **DHCP Release** — освобождение адреса клиентом;  
   – **DHCP Inform** — запрос дополнительных параметров без получения адреса.

3. **Какие параметры могут быть переданы в сообщениях DHCP?**  

   В DHCP-сообщениях могут передаваться различные сетевые параметры:  
   – IP-адрес клиента;  
   – маска подсети;  
   – адрес шлюза по умолчанию;  
   – адрес DNS-сервера;  
   – доменное имя;  
   – время аренды IP-адреса (lease time);  
   – адрес DHCP-сервера;  
   – дополнительные опции (например, NTP-сервер, TFTP-сервер и др.).  

   Эти параметры позволяют полностью автоматизировать настройку сетевого устройства.

4. **Что такое DNS?**  

   DNS (Domain Name System) — это распределённая система доменных имён, предназначенная для преобразования символьных доменных имён в IP-адреса.  

   Она позволяет пользователям обращаться к ресурсам сети по удобным именам (например, www.example.com) вместо числовых IP-адресов.  

   DNS работает по клиент-серверной модели и использует иерархическую структуру доменов.

5. **Какие типы записи описания ресурсов есть в DNS и для чего они используются?**  

   Основные типы DNS-записей:  
   – **A-запись (A Record)** — сопоставляет доменное имя с IPv4-адресом;  
   – **AAAA-запись** — сопоставляет имя с IPv6-адресом;  
   – **CNAME (Canonical Name)** — задаёт псевдоним для другого доменного имени;  
   – **MX (Mail Exchange)** — указывает почтовый сервер для домена;  
   – **NS (Name Server)** — определяет DNS-серверы, обслуживающие домен;  
   – **PTR (Pointer)** — используется для обратного разрешения IP-адреса в имя;  
   – **TXT** — содержит произвольную текстовую информацию (например, для SPF или проверки домена).  

   Эти записи обеспечивают работу системы доменных имён и позволяют различным сервисам корректно функционировать в сети.
